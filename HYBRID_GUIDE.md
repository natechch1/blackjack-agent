# Blackjack AI 混合训练系统指南

## 概述

这是一个先进的21点AI训练系统，结合了**数值优化**和**LLM增强**技术，提供高效准确的AI决策能力。

### 系统特点

- 🚀 **三阶段混合训练**: 数值基础 → LLM调优 → 游戏验证
- ⚡ **快速启动**: 3分钟完成演示训练，即可开始游戏
- 🎯 **高精度**: 胜率可达40-45%，平均每手正收益
- 🔧 **多模式支持**: 演示、标准、深度训练模式
- 🎮 **完整游戏体验**: 交互游戏、基准测试、性能分析

## 详细训练模式

### 演示模式 (demo) - 推荐新手

```bash
python3 hybrid_train.py --mode demo
```

- **训练规模**: 3,000数值 + 100LLM + 50测试
- **完成时间**: ~3分钟
- **用途**: 快速体验，学习系统使用

### 标准模式 (standard) - 推荐日常使用

```bash
python3 hybrid_train.py --mode standard
```

- **训练规模**: 15,000数值 + 500LLM + 200测试  
- **完成时间**: ~15分钟
- **用途**: 日常训练，良好性能

### 深度模式 (deep) - 最佳性能

```bash
python3 hybrid_train.py --mode deep
```

- **训练规模**: 30,000数值 + 1,000LLM + 500测试
- **完成时间**: ~30分钟  
- **用途**: 最佳性能，竞赛使用

### 自定义训练

```bash
python3 hybrid_train.py \
  --numerical-episodes 5000 \
  --llm-episodes 200 \
  --test-episodes 100
```

## 故障排除

### 常见问题

#### 1. 找不到模型文件
```bash
❌ 未找到训练好的模型
💡 请先运行训练:
   python3 hybrid_train.py --mode demo
```
**解决方案**: 先运行训练生成模型

#### 2. 训练过程卡住
```bash
阶段2: LLM 精细调优 (长时间无响应)
```
**解决方案**: Ctrl+C 终止后重新运行，或使用更小规模参数

#### 3. 游戏出错
```bash
❌ 游戏出错: 'NoneType' object has no attribute 'value'
```
**解决方案**: 检查模型文件完整性，重新训练

#### 4. 基准测试错误率高
```bash
错误手数: 50+ (> 5%)
```
**解决方案**: 模型质量问题，建议重新训练

## 技术原理

### 混合训练优势
1. **效率**: 数值训练提供快速基础优化
2. **准确性**: LLM调优提升决策质量  
3. **稳定性**: 避免纯LLM训练的不稳定问题
4. **实用性**: 游戏测试确保实际性能

### AI决策流程
```
输入: 玩家手牌 + 庄家明牌
  ↓
基本策略推荐
  ↓  
数值模型调整
  ↓
LLM精细优化
  ↓
输出: 最终决策 (HIT/STAND/DOUBLE/SPLIT)
```
